In mathematics <https://en.wikipedia.org/wiki/Mathematics> and computing
<https://en.wikipedia.org/wiki/Computing>, *hexadecimal* (also *base
16*, or *hex*) is a positional system
<https://en.wikipedia.org/wiki/Numeral_system#Positional_systems_in_detail>
that represents numbers using a base
<https://en.wikipedia.org/wiki/Radix> of 16. Unlike the common way
<https://en.wikipedia.org/wiki/Decimal> of representing numbers with ten
symbols, it uses sixteen distinct symbols, most often the symbols
"0"–"9" to represent values zero to nine, and "A"–"F" (or alternatively
"a"–"f") to represent values ten to fifteen.

Hexadecimal numerals are widely used by computer system designers and
programmers, as they provide a human-friendly representation of
binary-coded <https://en.wikipedia.org/wiki/Binary_code> values. Each
hexadecimal digit represents four binary digits
<https://en.wikipedia.org/wiki/Bit>, also known as a nibble
<https://en.wikipedia.org/wiki/Nibble>, which is half a byte
<https://en.wikipedia.org/wiki/Byte>. For example, a single byte can
have values ranging from 00000000 to 11111111 in binary form, which can
be conveniently represented as 00 to FF in hexadecimal.

In mathematics, a subscript is typically used to specify the base, also
known as the radix <https://en.wikipedia.org/wiki/Radix>. For example,
the decimal value 10,995 would be expressed in hexadecimal as 2AF3_16 .
In programming, a number of notations are used to support hexadecimal
representation, usually involving a prefix or suffix. The prefix |0x| is
used in C <https://en.wikipedia.org/wiki/C_(programming_language)> and
related languages, which would denote this value by |0x2AF3|.

Hexadecimal is used in the transfer encoding *Base16*, in which each
byte of the plaintext <https://en.wikipedia.org/wiki/Plaintext> is
broken into two 4-bit values and represented by two hexadecimal digits.

In contexts where the base <https://en.wikipedia.org/wiki/Radix> is not
clear, hexadecimal numbers can be ambiguous and confused with numbers
expressed in other bases. There are several conventions for expressing
values unambiguously. A numerical subscript (itself written in decimal)
can give the base explicitly: 159_10 is decimal 159; 159_16 is
hexadecimal 159, which is equal to 345_10 . Some authors prefer a text
subscript, such as 159_decimal and 159_hex , or 159_d and 159_h .

In linear text systems, such as those used in most computer programming
environments, a variety of methods have arisen:

  * In URIs <https://en.wikipedia.org/wiki/URI> (including URLs
    <https://en.wikipedia.org/wiki/URL>), character codes
    <https://en.wikipedia.org/wiki/Character_encoding> are written as
    hexadecimal pairs prefixed with |%|:
    |http://www.example.com/name%20with%20spaces| where |%20| is the
    code for the space (blank)
    <https://en.wikipedia.org/wiki/Space_(punctuation)#Space_characters_and_digital_typography>
    character, ASCII <https://en.wikipedia.org/wiki/ASCII> code point 20
    in hex, 32 in decimal.
  * In XML <https://en.wikipedia.org/wiki/XML> and XHTML
    <https://en.wikipedia.org/wiki/XHTML>, characters can be expressed
    as hexadecimal numeric character references
    <https://en.wikipedia.org/wiki/Numeric_character_reference> using
    the notation |&#x/code/;|, where the /x/ denotes that /code/ is a
    hex code point (of 1- to 6-digits) assigned to the character in the
    Unicode <https://en.wikipedia.org/wiki/Unicode> standard. Thus
    |&#x2019;| represents the right single quotation mark (’), Unicode
    code point number 2019 in hex, 8217 (thus |&#8217;| in decimal).^[1]
    <#cite_note-1>
  * In the Unicode <https://en.wikipedia.org/wiki/Unicode> standard, a
    character value is represented with |U+| followed by the hex value,
    e.g. |U+20AC| is the Euro sign
    <https://en.wikipedia.org/wiki/Euro_sign> (€).
  * Color references <https://en.wikipedia.org/wiki/Web_colors> in HTML,
    CSS <https://en.wikipedia.org/wiki/Cascading_Style_Sheets> and X
    Window <https://en.wikipedia.org/wiki/X_window_system> can be
    expressed with six hexadecimal digits (two each for the red, green
    and blue components, in that order) prefixed with |#|: white, for
    example, is represented as |#FFFFFF|.^[2] <#cite_note-2> CSS also
    allows 3-hexdigit abbreviations with one hexdigit per component:
    #FA3 abbreviates #FFAA33 (a golden orange:     ).
  * Unix <https://en.wikipedia.org/wiki/*nix> (and related) shells, AT&T
    <https://en.wikipedia.org/wiki/AT%26T_Corporation> assembly language
    and likewise the C programming language
    <https://en.wikipedia.org/wiki/C_(programming_language)> (and its
    syntactic descendants such as C++
    <https://en.wikipedia.org/wiki/C%2B%2B>, C#
    <https://en.wikipedia.org/wiki/C_Sharp_(programming_language)>, D
    <https://en.wikipedia.org/wiki/D_(programming_language)>, Java
    <https://en.wikipedia.org/wiki/Java_(programming_language)>,
    JavaScript <https://en.wikipedia.org/wiki/JavaScript>, Python
    <https://en.wikipedia.org/wiki/Python_(programming_language)> and
    Windows PowerShell
    <https://en.wikipedia.org/wiki/Windows_PowerShell>) use the prefix
    |0x| for numeric constants represented in hex: |0x5A3|. Character
    and string constants may express character codes in hexadecimal with
    the prefix |\x| followed by two hex digits: |'\x1B'| represents the
    Esc <https://en.wikipedia.org/wiki/Escape_character> control
    character; |"\x1B[0m\x1B[25;1H"| is a string containing 11
    characters (plus a trailing NUL to mark the end of the string) with
    two embedded Esc characters.^[3] <#cite_note-3> To output an integer
    as hexadecimal with the printf
    <https://en.wikipedia.org/wiki/Printf> function family, the format
    conversion code |%X| or |%x| is used.
  * In MIME <https://en.wikipedia.org/wiki/MIME> (e-mail extensions)
    quoted-printable <https://en.wikipedia.org/wiki/Quoted-printable>
    encoding, characters that cannot be represented as literal ASCII
    <https://en.wikipedia.org/wiki/ASCII> characters are represented by
    their codes as two hexadecimal digits (in ASCII) prefixed by an
    /equal to/ sign |=|, as in |Espa=F1a| to send "España" (Spain).
    (Hexadecimal F1, equal to decimal 241, is the code number for the
    lower case n with tilde in the ISO/IEC 8859-1 character set.^[4]
    <#cite_note-4> )
  * In Intel-derived assembly languages
    <https://en.wikipedia.org/wiki/Assembly_language> and Modula-2,^[5]
    <#cite_note-5> hexadecimal is denoted with a suffixed H or h: |FFh|
    or |05A3H|. Some implementations require a leading zero when the
    first hexadecimal digit character is not a decimal digit, so one
    would write |0FFh| instead of |FFh|
  * Other assembly languages (6502
    <https://en.wikipedia.org/wiki/MOS_Technology_6502>, Motorola
    <https://en.wikipedia.org/wiki/Motorola>), Pascal
    <https://en.wikipedia.org/wiki/Pascal_(programming_language)>,
    Delphi <https://en.wikipedia.org/wiki/Object_Pascal>, some versions
    of BASIC <https://en.wikipedia.org/wiki/BASIC> (Commodore
    <https://en.wikipedia.org/wiki/Commodore_BASIC>), GameMaker Language
    <https://en.wikipedia.org/wiki/GameMaker_Studio>, Godot
    <https://en.wikipedia.org/wiki/Godot_(game_engine)> and Forth
    <https://en.wikipedia.org/wiki/Forth_(programming_language)> use |$|
    as a prefix: |$5A3|.
  * Some assembly languages (Microchip) use the notation |H'ABCD'| (for
    ABCD_16 ). Similarly, Fortran 95
    <https://en.wikipedia.org/wiki/Fortran_95_language_features> uses
    Z'ABCD'.
  * Ada <https://en.wikipedia.org/wiki/Ada_(programming_language)> and
    VHDL <https://en.wikipedia.org/wiki/VHDL> enclose hexadecimal
    numerals in based "numeric quotes": |16#5A3#|. For bit vector
    constants VHDL <https://en.wikipedia.org/wiki/VHDL> uses the
    notation |x"5A3"|.^[6] <#cite_note-6>
  * Verilog <https://en.wikipedia.org/wiki/Verilog> represents
    hexadecimal constants in the form |8'hFF|, where 8 is the number of
    bits in the value and FF is the hexadecimal constant.
  * The Smalltalk <https://en.wikipedia.org/wiki/Smalltalk> language
    uses the prefix |16r|: |16r5A3|
  * PostScript <https://en.wikipedia.org/wiki/PostScript> and the Bourne
    shell <https://en.wikipedia.org/wiki/Bourne_shell> and its
    derivatives denote hex with prefix |16#|: |16#5A3|. For PostScript,
    binary data (such as image pixels
    <https://en.wikipedia.org/wiki/Pixel>) can be expressed as
    unprefixed consecutive hexadecimal pairs: |AA213FD51B3801043FBC|...
  * Common Lisp <https://en.wikipedia.org/wiki/Common_Lisp> uses the
    prefixes |#x| and |#16r|. Setting the variables *read-base*^[7]
    <#cite_note-7> and *print-base*^[8] <#cite_note-8> to 16 can also be
    used to switch the reader and printer of a Common Lisp system to
    Hexadecimal number representation for reading and printing numbers.
    Thus Hexadecimal numbers can be represented without the #x or #16r
    prefix code, when the input or output base has been changed to 16.
  * MSX BASIC <https://en.wikipedia.org/wiki/MSX_BASIC>,^[9]
    <#cite_note-9> QuickBASIC
    <https://en.wikipedia.org/wiki/QuickBASIC>, FreeBASIC
    <https://en.wikipedia.org/wiki/FreeBASIC> and Visual Basic
    <https://en.wikipedia.org/wiki/Visual_Basic> prefix hexadecimal
    numbers with |&H|: |&H5A3|
  * BBC BASIC <https://en.wikipedia.org/wiki/BBC_BASIC> and Locomotive
    BASIC <https://en.wikipedia.org/wiki/Locomotive_BASIC> use |&| for
    hex.^[10] <#cite_note-10>
  * TI-89 <https://en.wikipedia.org/wiki/TI-89> and 92 series uses a
    |0h| prefix: |0h5A3|
  * ALGOL 68 <https://en.wikipedia.org/wiki/ALGOL_68> uses the prefix
    |16r| to denote hexadecimal numbers: |16r5a3|. Binary, quaternary
    (base-4) and octal numbers can be specified similarly.
  * The most common format for hexadecimal on IBM mainframes (zSeries
    <https://en.wikipedia.org/wiki/ZSeries>) and midrange computers (IBM
    System i <https://en.wikipedia.org/wiki/IBM_System_i>) running the
    traditional OS's (zOS <https://en.wikipedia.org/wiki/Z/OS>, zVSE
    <https://en.wikipedia.org/wiki/VSE_(operating_system)>, zVM
    <https://en.wikipedia.org/wiki/Z/VM>, TPF
    <https://en.wikipedia.org/wiki/Transaction_Processing_Facility>, IBM
    i <https://en.wikipedia.org/wiki/IBM_i>) is |X'5A3'|, and is used in
    Assembler, PL/I <https://en.wikipedia.org/wiki/PL/I>, COBOL
    <https://en.wikipedia.org/wiki/COBOL>, JCL
    <https://en.wikipedia.org/wiki/Job_Control_Language>, scripts,
    commands and other places. This format was common on other (and now
    obsolete) IBM systems as well. Occasionally quotation marks were
    used instead of apostrophes.
  * Donald Knuth <https://en.wikipedia.org/wiki/Donald_Knuth> introduced
    the use of a particular typeface to represent a particular radix in
    his book /The TeXbook/.^[11] <#cite_note-11> Hexadecimal
    representations are written there in a typewriter typeface: 5A3
  * Any IPv6 address <https://en.wikipedia.org/wiki/IPv6_address> can be
    written as eight groups of four hexadecimal digits (sometimes called
    hextets <https://en.wikipedia.org/wiki/Hextet_(computing)>), where
    each group is separated by a colon (|:|). This, for example, is a
    valid IPv6 address: 2001:0db8:85a3:0000:0000:8a2e:0370:7334; this
    can be abbreviated as 2001:db8:85a3::8a2e:370:7334. By contrast,
    IPv4 addresses <https://en.wikipedia.org/wiki/IPv4_address> are
    usually written in decimal.
  * Globally unique identifiers
    <https://en.wikipedia.org/wiki/Globally_unique_identifier> are
    written as thirty-two hexadecimal digits, often in unequal
    hyphen-separated groupings, for example
    |{3F2504E0-4F89-41D3-9A0C-0305E82C3301}|.

There is no universal convention to use lowercase or uppercase for the
letter digits, and each is prevalent or preferred in particular
environments by community standards or convention.

The use of the letters /A/ through /F/ to represent the digits above 9
was not universal in the early history of computers.

  * During the 1950s, some installations^[/which?
    <https://en.wikipedia.org/wiki/Wikipedia:Avoid_weasel_words>/]
    favored using the digits 0 through 5 with an overline
    <https://en.wikipedia.org/wiki/Overline> to denote the values 10–15
    as 0, 1, 2, 3, 4 and 5.
  * The SWAC <https://en.wikipedia.org/wiki/SWAC_(computer)> (1950)^[13]
    <#cite_note-Savard_2018_CA-13> and Bendix G-15
    <https://en.wikipedia.org/wiki/Bendix_G-15> (1956)^[14]
    <#cite_note-Bendix-14> ^[13] <#cite_note-Savard_2018_CA-13>
    computers used the lowercase letters /u/, /v/, /w/, /x/, /y/ and /z/
    for the values 10 to 15.
  * The ILLIAC I <https://en.wikipedia.org/wiki/ILLIAC_I> (1952)
    computer used the uppercase letters /K/, /S/, /N/, /J/, /F/ and /L/
    for the values 10 to 15.^[15] <#cite_note-Illiac-I-15> ^[13]
    <#cite_note-Savard_2018_CA-13>

      Verbal and digital representations[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=5>]

There are no traditional numerals to represent the quantities from ten
to fifteen – letters are used as a substitute – and most European
<https://en.wikipedia.org/wiki/Europe> languages lack non-decimal names
for the numerals above ten. Even though English has names for several
non-decimal powers (/pair <https://en.wikipedia.org/wiki/2_(number)>/
for the first binary
<https://en.wikipedia.org/wiki/Binary_numeral_system> power, /score
<https://en.wikipedia.org/wiki/20_(number)>/ for the first vigesimal
<https://en.wikipedia.org/wiki/Vigesimal> power, /dozen
<https://en.wikipedia.org/wiki/Dozen>/, /gross
<https://en.wikipedia.org/wiki/Gross_(unit)>/ and /great gross
<https://en.wikipedia.org/wiki/Great_gross>/ for the first three
duodecimal <https://en.wikipedia.org/wiki/Duodecimal> powers), no
English name describes the hexadecimal powers (decimal 16, 256, 4096,
65536, ... ). Some people read hexadecimal numbers digit by digit like a
phone number, or using the NATO phonetic alphabet
<https://en.wikipedia.org/wiki/ICAO_spelling_alphabet>, the Joint
Army/Navy Phonetic Alphabet
<https://en.wikipedia.org/wiki/Joint_Army/Navy_Phonetic_Alphabet>, or a
similar ad hoc system. In the wake of the adoption of hexadecimal among
IBM System/360 <https://en.wikipedia.org/wiki/IBM_System/360>
programmers, Robert A. Magnuson suggested in 1968 in Datamation
<https://en.wikipedia.org/wiki/Datamation> Magazine a pronunciation
guide that gave short names to the letters of hexadecimal - for
instance, "A" was pronounced "ann", B "bet", C "chris", etc.^[19]
<#cite_note-19> Another naming system was invented independently by Tim
Babb in 2015.^[20] <#cite_note-20> An additional naming system has been
published online by S. R. Rogers in 2007^[21] <#cite_note-21> that tries
to make the verbal representation distiguishable in any case, even when
the actual number does not contain numbers A-F. Examples are listed in
the tables below.

      Signs[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=6>]

The hexadecimal system can express negative numbers the same way as in
decimal: −2A to represent −42_10 and so on.

Hexadecimal can also be used to express the exact bit patterns used in
the processor <https://en.wikipedia.org/wiki/Central_processing_unit>,
so a sequence of hexadecimal digits may represent a signed
<https://en.wikipedia.org/wiki/Signedness> or even a floating point
<https://en.wikipedia.org/wiki/Floating_point> value. This way, the
negative number −42_10 can be written as FFFF FFD6 in a 32-bit CPU
register <https://en.wikipedia.org/wiki/Processor_register> (in
two's-complement <https://en.wikipedia.org/wiki/Two%27s-complement>), as
C228 0000 in a 32-bit FPU
<https://en.wikipedia.org/wiki/Floating_point_unit> register or
C045 0000 0000 0000 in a 64-bit FPU register (in the IEEE floating-point
standard <https://en.wikipedia.org/wiki/IEEE_floating-point_standard>).


      Hexadecimal exponential notation[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=7>]

Just as decimal numbers can be represented in exponential notation
<https://en.wikipedia.org/wiki/Exponential_notation>, so too can
hexadecimal numbers. By convention, the letter /P/ (or /p/, for "power")
represents /times two raised to the power of/, whereas /E/ (or /e/)
serves a similar purpose in decimal as part of the E notation
<https://en.wikipedia.org/wiki/E_notation>. The number after the /P/ is
/decimal/ and represents the /binary/ exponent.

Usually the number is normalised so that the leading hexadecimal digit
is 1 (unless the value is exactly 0).

Example: 1.3DEp42 represents 1.3DE_16  × 2^42 .

Hexadecimal exponential notation is required by the IEEE 754-2008
<https://en.wikipedia.org/wiki/IEEE_754-2008> binary floating-point
standard. This notation can be used for floating-point literals in the
C99 <https://en.wikipedia.org/wiki/C99> edition of the C programming
language <https://en.wikipedia.org/wiki/C_(programming_language)>.^[23]
<#cite_note-23> Using the /%a/ or /%A/ conversion specifiers, this
notation can be produced by implementations of the /printf
<https://en.wikipedia.org/wiki/Printf>/ family of functions following
the C99 specification^[24] <#cite_note-Rationale_2003_C-24> and Single
Unix Specification
<https://en.wikipedia.org/wiki/Single_UNIX_Specification> (IEEE Std
1003.1) POSIX <https://en.wikipedia.org/wiki/POSIX> standard.^[25]
<#cite_note-printf_2013-25>


    Conversion[edit
    <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=8>]


      Binary conversion[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=9>]

Most computers manipulate binary data, but it is difficult for humans to
work with the large number of digits for even a relatively small binary
number. Although most humans are familiar with the base 10 system, it is
much easier to map binary to hexadecimal than to decimal because each
hexadecimal digit maps to a whole number of bits (4_10 ). This example
converts 1111_2 to base ten. Since each position
<https://en.wikipedia.org/wiki/Positional_notation> in a binary numeral
can contain either a 1 or a 0, its value may be easily determined by its
position from the right:

  * 0001_2 = 1_10
  * 0010_2 = 2_10
  * 0100_2 = 4_10
  * 1000_2 = 8_10 

Therefore:

1111_2 	= 8_10 + 4_10 + 2_10 + 1_10
  	= 15_10

With little practice, mapping 1111_2 to F_16 in one step becomes easy:
see table in written representation <#Written_representation>. The
advantage of using hexadecimal rather than decimal increases rapidly
with the size of the number. When the number becomes large, conversion
to decimal is very tedious. However, when mapping to hexadecimal, it is
trivial to regard the binary string as 4-digit groups and map each to a
single hexadecimal digit.

This example shows the conversion of a binary number to decimal, mapping
each digit to the decimal value, and adding the results.

(01011110101101010010)_2 	= 262144_10 + 65536_10 + 32768_10 + 16384_10 +
8192_10 + 2048_10 + 512_10 + 256_10 + 64_10 + 16_10 + 2_10
  	= 387922_10

Compare this to the conversion to hexadecimal, where each group of four
digits can be considered independently, and converted directly:

(01011110101101010010)_2 	= 	0101_  	1110_  	1011_  	0101_  	0010_2
  	= 	5 	E 	B 	5 	2_16
  	= 	5EB52_16

The conversion from hexadecimal to binary is equally direct.


      Other simple conversions[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=10>]

Although quaternary
<https://en.wikipedia.org/wiki/Quaternary_numeral_system> (base 4) is
little used, it can easily be converted to and from hexadecimal or
binary. Each hexadecimal digit corresponds to a pair of quaternary
digits and each quaternary digit corresponds to a pair of binary digits.
In the above example 5 E B 5 2_16 = 11 32 23 11 02_4 .

The octal <https://en.wikipedia.org/wiki/Octal> (base 8) system can also
be converted with relative ease, although not quite as trivially as with
bases 2 and 4. Each octal digit corresponds to three binary digits,
rather than four. Therefore we can convert between octal and hexadecimal
via an intermediate conversion to binary followed by regrouping the
binary digits in groups of either three or four.


      Division-remainder in source base[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=11>]

As with all bases there is a simple algorithm
<https://en.wikipedia.org/wiki/Algorithm> for converting a
representation of a number to hexadecimal by doing integer division and
remainder operations in the source base. In theory, this is possible
from any base, but for most humans only decimal and for most computers
only binary (which can be converted by far more efficient methods) can
be easily handled with this method.

Let d be the number to represent in hexadecimal, and the series h_i
h_i−1 ...h_2 h_1 be the hexadecimal digits representing the number.

 1. i ← 1
 2. h_i ← d mod 16
 3. d ← (d − h_i ) / 16
 4. If d = 0 (return series h_i ) else increment i and go to step 2

"16" may be replaced with any other base that may be desired.

The following is a JavaScript <https://en.wikipedia.org/wiki/JavaScript>
implementation of the above algorithm for converting any number to a
hexadecimal in String representation. Its purpose is to illustrate the
above algorithm. To work with data seriously, however, it is much more
advisable to work with bitwise operators
<https://en.wikipedia.org/wiki/Bitwise_operators>.

function toHex(d) {
  var r = d % 16;
  if (d - r == 0) {
    return toChar(r);
  }
  return toHex( (d - r)/16 ) + toChar(r);
}

function toChar(n) {
  const alpha = "0123456789ABCDEF";
  return alpha.charAt(n);
}


      Conversion through addition and multiplication[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=12>]

<https://en.wikipedia.org/wiki/File:Hexadecimal_multiplication_table.svg>
<https://en.wikipedia.org/wiki/File:Hexadecimal_multiplication_table.svg>
A hexadecimal multiplication table
<https://en.wikipedia.org/wiki/Multiplication_table>

It is also possible to make the conversion by assigning each place in
the source base the hexadecimal representation of its place value —
before carrying out multiplication and addition to get the final
representation. For example, to convert the number B3AD to decimal, one
can split the hexadecimal number into its digits: B (11_10 ), 3 (3_10 ),
A (10_10 ) and D (13_10 ), and then get the final result by multiplying
each decimal representation by 16^/p/ (/p/ being the corresponding hex
digit position, counting from right to left, beginning with 0). In this
case, we have that:

B3AD = (11 × 16^3 ) + (3 × 16^2 ) + (10 × 16^1 ) + (13 × 16^0 )

which is 45997 in base 10.


      Tools for conversion[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=13>]

Most modern computer systems with graphical user interfaces
<https://en.wikipedia.org/wiki/Graphical_user_interface> provide a
built-in calculator utility capable of performing conversions between
the various radices, and in most cases would include the hexadecimal as
well.

In Microsoft Windows <https://en.wikipedia.org/wiki/Microsoft_Windows>,
the Calculator <https://en.wikipedia.org/wiki/Calculator_(Windows)>
utility can be set to Scientific mode (called Programmer mode in some
versions), which allows conversions between radix 16 (hexadecimal), 10
(decimal), 8 (octal <https://en.wikipedia.org/wiki/Octal>) and 2 (binary
<https://en.wikipedia.org/wiki/Binary_numeral_system>), the bases most
commonly used by programmers. In Scientific Mode, the on-screen numeric
keypad <https://en.wikipedia.org/wiki/Numeric_keypad> includes the
hexadecimal digits A through F, which are active when "Hex" is selected.
In hex mode, however, the Windows Calculator supports only integers.


    Elementary arithmetic[edit
    <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=14>]

Elementary operations such additions, subtractions, multiplications and
divisions can be carried out indirectly through conversion to an
alternate numeral system <https://en.wikipedia.org/wiki/Numeral_system>,
such as the decimal system, since it's the most commonly adopted system,
or the binary system, since each hex digit corresponds to four binary
digits,

Alternatively, one can also perform elementary operations directly
within the hex system itself — by relying on its addition/multiplication
tables and its corresponding standard algorithms such as long division
<https://en.wikipedia.org/wiki/Long_division> and the traditional
subtraction algorithm.^[26] <#cite_note-26>

      Rational numbers[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=16>]

As with other numeral systems, the hexadecimal system can be used to
represent rational numbers
<https://en.wikipedia.org/wiki/Rational_number>, although repeating
expansions <https://en.wikipedia.org/wiki/Repeating_decimal> are common
since sixteen (10_16 ) has only a single prime factor; two.

For any base, 0.1 (or "1/10") is always equivalent to one divided by the
representation of that base value in its own number system. Thus,
whether dividing one by two for binary
<https://en.wikipedia.org/wiki/Binary_numeral_system> or dividing one by
sixteen for hexadecimal, both of these fractions are written as |0.1|.
Because the radix 16 is a perfect square
<https://en.wikipedia.org/wiki/Square_number> (4^2 ), fractions
expressed in hexadecimal have an odd period much more often than decimal
ones, and there are no cyclic numbers
<https://en.wikipedia.org/wiki/Cyclic_number> (other than trivial single
digits). Recurring digits are exhibited when the denominator in lowest
terms has a prime factor <https://en.wikipedia.org/wiki/Prime_factor>
not found in the radix; thus, when using hexadecimal notation, all
fractions with denominators that are not a power of two
<https://en.wikipedia.org/wiki/Power_of_two> result in an infinite
string of recurring digits (such as thirds and fifths). This makes
hexadecimal (and binary) less convenient than decimal
<https://en.wikipedia.org/wiki/Decimal> for representing rational
numbers since a larger proportion lie outside its range of finite
representation.

All rational numbers finitely representable in hexadecimal are also
finitely representable in decimal, duodecimal
<https://en.wikipedia.org/wiki/Duodecimal> and sexagesimal
<https://en.wikipedia.org/wiki/Sexagesimal>: that is, any hexadecimal
number with a finite number of digits also has a finite number of digits
when expressed in those other bases. Conversely, only a fraction of
those finitely representable in the latter bases are finitely
representable in hexadecimal. For example, decimal 0.1 corresponds to
the infinite recurring representation 0.19 in hexadecimal. However,
hexadecimal is more efficient than duodecimal and sexagesimal for
representing fractions with powers of two in the denominator. For
example, 0.0625_10 (one sixteenth) is equivalent to 0.1_16 , 0.09_12 ,
and 0;3,45_60 .

    Cultural[edit
    <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=19>]


      Etymology[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=20>]

The word /hexadecimal/ is composed of /hexa-/, derived from the Greek
<https://en.wikipedia.org/wiki/Greek_language> ἕξ (hex) for /six/, and
/-decimal/, derived from the Latin <https://en.wikipedia.org/wiki/Latin>
for /tenth/. Webster's Third New International online derives
/hexadecimal/ as an alteration of the all-Latin /sexadecimal/ (which
appears in the earlier Bendix documentation). The earliest date attested
for /hexadecimal/ in Merriam-Webster Collegiate online is 1954, placing
it safely in the category of international scientific vocabulary
<https://en.wikipedia.org/wiki/International_scientific_vocabulary>
(ISV). It is common in ISV to mix Greek and Latin combining forms
<https://en.wikipedia.org/wiki/Combining_form> freely. The word
/sexagesimal <https://en.wikipedia.org/wiki/Sexagesimal>/ (for base 60)
retains the Latin prefix. Donald Knuth
<https://en.wikipedia.org/wiki/Donald_Knuth> has pointed out that the
etymologically correct term is /senidenary/ (or possibly, /sedenary/),
from the Latin term for /grouped by 16/. (The terms /binary/, /ternary/
and /quaternary/ are from the same Latin construction, and the
etymologically correct terms for /decimal/ and /octal/ arithmetic are
/denary/ and /octonary/, respectively.)^[27] <#cite_note-27> Alfred B.
Taylor used /senidenary/ in his mid-1800s work on alternative number
bases, although he rejected base 16 because of its "incommodious number
of digits".^[28] <#cite_note-28> ^[29] <#cite_note-29> Schwartzman notes
that the expected form from usual Latin phrasing would be /sexadecimal/,
but computer hackers would be tempted to shorten that word to
/sex/.^[30] <#cite_note-30> The etymologically
<https://en.wikipedia.org/wiki/Etymology> proper Greek
<https://en.wikipedia.org/wiki/Greek_language> term would be
/hexadecadic/ / /ἑξαδεκαδικός/ / /hexadekadikós/ (although in Modern
Greek <https://en.wikipedia.org/wiki/Modern_Greek>, /decahexadic/ /
/δεκαεξαδικός/ / /dekaexadikos/ is more commonly used).


      Use in Chinese culture[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=21>]

The traditional Chinese units of measurement
<https://en.wikipedia.org/wiki/Chinese_units_of_measurement> were
base-16. For example, one jīn (斤) in the old system equals sixteen
taels <https://en.wikipedia.org/wiki/Tael>. The suanpan
<https://en.wikipedia.org/wiki/Suanpan> (Chinese abacus
<https://en.wikipedia.org/wiki/Abacus>) can be used to perform
hexadecimal calculations such as additions and subtractions.^[31]
<#cite_note-31>


      Primary numeral system[edit
      <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=22>]

As with the duodecimal <https://en.wikipedia.org/wiki/Duodecimal>
system, there have been occasional attempts to promote hexadecimal as
the preferred numeral system. These attempts often propose specific
pronunciation and symbols for the individual numerals.^[32]
<#cite_note-32> Some proposals unify standard measures so that they are
multiples of 16.^[33] <#cite_note-33> ^[34] <#cite_note-34> ^[35]
<#cite_note-nystrom-35>

An example of unified standard measures is hexadecimal time
<https://en.wikipedia.org/wiki/Hexadecimal_time>, which subdivides a day
by 16 so that there are 16 "hexhours" in a day.^[35]
<#cite_note-nystrom-35>


    Base16 (Transfer encoding)[edit
    <https://en.wikipedia.org/w/index.php?title=Hexadecimal&action=edit&section=23>]

*Base16* (as a proper name without a space) can also refer to a binary
to text encoding <https://en.wikipedia.org/wiki/Binary_to_text_encoding>
belonging to the same family as Base32
<https://en.wikipedia.org/wiki/Base32>, Base58
<https://en.wikipedia.org/wiki/Base58>, and Base64
<https://en.wikipedia.org/wiki/Base64>.

In this case, data is broken into 4-bit sequences, and each value
(between 0 and 15 inclusively) is encoded using 16 symbols from the
ASCII <https://en.wikipedia.org/wiki/ASCII> character set. Although any
16 symbols from the ASCII character set can be used, in practice the
ASCII digits '0'-'9' and the letters 'A'-'F' (or the lowercase 'a'-'f')
are always chosen in order to align with standard written notation for
hexadecimal numbers.

There are several advantages of Base16 encoding:

  * Most programming languages already have facilities to parse
    ASCII-encoded hexadecimal
  * Being exactly half a byte, 4-bits is easier to process than the 5 or
    6 bits of Base32 and Base64 respectively
  * The symbols 0-9 and A-F are universal in hexadecimal notation, so it
    is easily understood at a glance without needing to rely on a symbol
    lookup table
  * Many CPU architectures have dedicated instructions that allow access
    to a half-byte (otherwise known as a "Nibble
    <https://en.wikipedia.org/wiki/Nibble>"), making it more efficient
    in hardware than Base32 and Base64

The main disadvantages of Base16 encoding are:

  * Space efficiency is only 50%, since each 4-bit value from the
    original data will be encoded as an 8-bit byte. In contrast, Base32
    and Base64 encodings have a space efficiency of 63% and 75%
    respectively.
  * Possible added complexity of having to accept both uppercase and
    lowercase letters

Support for Base16 encoding is ubiquitous in modern computing. It is the
basis for the W3C
<https://en.wikipedia.org/wiki/World_Wide_Web_Consortium> standard for
URL Percent Encoding <https://en.wikipedia.org/wiki/Percent-encoding>,
where a character is replaced with a percent sign "%" and its
Base16-encoded form. Most modern programming languages directly include
support for formatting and parsing Base16-encoded numbers.
